# just-the-docs 테마 적용 가이드

## 0. 준비물
이 압축 파일의 내용물이 새 블로그 소스 전체입니다. (글 9개 + 이미지 포함 이전 완료)

## 1. 기존 블로그 백업
```bash
cd hyunbenny.github.io
git checkout master
git checkout -b backup-minimal-mistakes
git push origin backup-minimal-mistakes
git checkout master
```

## 2. master 내용 교체
```bash
# master에서 기존 파일 전부 삭제 후 (숨김파일 .git 제외)
git rm -rf . && git clean -fdx
# 압축 해제한 새 파일들을 저장소 루트에 복사한 뒤
git add -A
git commit -m "테마 변경: minimal-mistakes → just-the-docs"
git push origin master
```

## 3. GitHub Pages 배포 방식 변경 (중요!)
저장소 **Settings → Pages → Build and deployment → Source**를
`Deploy from a branch` → **`GitHub Actions`** 로 변경합니다.
(just-the-docs 최신 버전은 기본 Pages 빌드로는 설치되지 않아 Actions 배포가 필요합니다)

푸시하면 Actions 탭에서 "Deploy Jekyll site to Pages" 워크플로가 돌고,
성공하면 https://hyunbenny.github.io 에 반영됩니다.

## 4. 확인 사항
- 기존 글 주소는 그대로 유지됩니다 (예: /spring/멀티모듈-프로젝트-세팅/)
- 사이드바: Home / 태그 / About / 카테고리 5개(Spring·Database·Python·Anaconda·NodeJs)
- 사이드바 하단: 프로필 + 다크 모드 토글 (선택이 브라우저에 저장됨)
- 새 글 쓰기: docs/<카테고리>/ 아래에 md 추가, front matter는 기존 글 복사해서 수정
  (layout: post, title, parent, nav_order, date, tags, permalink)
- 새 카테고리: docs/<이름>/index.md 를 기존 카테고리 index.md 참고해 생성

## 5. 로컬에서 미리 보기 (선택)
```bash
bundle install
bundle exec jekyll serve
# http://localhost:4000
```

## 되돌리기
문제가 생기면 backup-minimal-mistakes 브랜치를 master로 되돌리고
Pages Source를 다시 `Deploy from a branch`로 바꾸면 됩니다.
